Protocol Design

How It Works

AAMN routes your traffic through multiple hops — each hop only knows the adjacent nodes.

1

Node Identity & Discovery

Each node generates an Ed25519 identity key and publishes itself to the Kademlia DHT. Peers are discovered without any central registry.

2

Noise IKpsk2 Handshake

Before any data flows, nodes perform a mutual X25519 key exchange via the Noise IKpsk2 pattern, establishing a forward-secret session with PSK verification.

3

Onion Circuit Construction

The sender chooses 3+ relay nodes and wraps the payload in nested ChaCha20-Poly1305 layers — one per hop. Each relay decrypts only its own layer.

4

Fixed-Cell Transmission

All packets are padded to exactly 512 bytes (cells), preventing traffic-analysis attacks that rely on packet size correlation.

5

Rate-Limited Relay

Token-bucket and sliding-window rate limiters at each node prevent DDoS amplification and further anonymize timing patterns.
